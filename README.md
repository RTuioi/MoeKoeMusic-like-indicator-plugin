# 歌曲收藏状态指示器
> 首先十分感谢「阿珏」及 萌え音 的[MoeKoeMusic](https://github.com/MoeKoeMusic/MoeKoeMusic) 项目。

MoeKoeMusic 插件,给播放栏和全屏歌词页的红心按钮加上收藏状态反馈——已收藏的歌曲，红心会跟着主题色亮起来，不用再去 Kg 看到底收藏没。

## 插件功能特性

- **底部播放栏红心**：当前歌曲在"我喜欢"歌单里时，红心变主题色
- **全屏歌词页红心**：同上，全屏歌词页那个红心按钮也跟着亮
- **切歌自动刷新**：换歌时自动判断收藏状态
- **收藏操作实时同步**：在播放栏点了收藏/取消收藏，红心状态立刻更新
- **跨音质匹配**：搜索、排行榜里播放的是高音质版本时，也能认出这首歌已收藏（标准音质 hash 对不上的时候，会主动查一遍音质信息）
- **登录/登出自动处理**：登出清空数据，登录重新拉取

## 食用方法

1. 插件市场安装或者下载插件zip将插件加载到 MoeKoeMusic 的插件系统中（推:设置 → 插件 → 安装插件 → 选择此插件 → 安装即可→刷新或重启）
2. 重启 MoeKoeMusic，或者进设置 → 插件管理 → 重新加载 → 页面刷新

文件结构：

```
LikeIndicator-Plugin/
├── manifest.json

```

## 测试指南

装好后播一首你已经收藏的歌，红心应该是主题色（默认粉/红那种），不是灰色。再播一首没收藏的，红心变回灰色。

## 测过哪些场景

下面这些场景测过，红心状态正确：

- 我的歌单（自己创建的歌单里播放）
- 收藏歌单（"我喜欢"歌单）
- 首页推荐的歌单
- 搜索界面（搜出来的歌点播放）
- 排行榜
- 歌手页（歌手详情里的歌）

## 没测的场景

- 私人推荐：主要是试了六七次抽不到我收藏的歌曲，暂时没法测。

## 工作原理


1. **拦截收藏请求**：你点红心收藏/取消收藏时，主程序会发请求给后端，插件监听，收藏了就立刻更新红心，不用下次刷新
2. **拉取收藏列表**：登录后从"我喜欢"歌单把所有歌曲 hash 拉下来缓存（缓存 1 小时），切歌时拿当前歌曲 hash 去比对
3. **跨音质匹配**：搜索/排行榜里播放的可能是高音质版本，hash 跟收藏列表里的标准音质 hash 对不上。这时候插件会主动调一次音质接口，把这首歌所有版本的 hash 都拿出来比对，只要有任何一个版本在收藏列表里，就判定为已收藏

## 出问题怎么反馈

提 issue：https://github.com/RTuioi/MoeKoeMusic-like-indicator-plugin

反馈时请说明：

1. 是哪首歌曲（歌名 + 歌手）最好是完整截图
2. 在哪个界面播放的（搜索、排行榜、歌单……）
3. 附上控制台日志(推:可在设置-系统-日志 中 导出日志,也可以在 MoeKoeMusic 里按 `Ctrl+Shift+I` 打开控制台)


## 兼容性

- MoeKoeMusic 1.6.7 及以上
- 不依赖其他或额外库

## 已知

- 初始化加载收藏列表期间（通常 1-2 秒），切到的歌可能短暂显示未收藏，列表加载完会自动修正
- 私人推荐场景未测试
- 依赖主程序的 `/privilege/lite` 接口做跨音质匹配，如果后端这个接口挂了，跨音质匹配会失效（直接 hash 匹配仍然有效）
